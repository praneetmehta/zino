# Documentation Archive

This directory contains historical documentation that has been superseded by the consolidated docs in the parent directories.

## Why Archive?

These documents were created during development to track specific features, fixes, and milestones. While they're no longer the primary documentation, they're preserved here for:

- **Historical reference** - Understanding how features evolved
- **Implementation details** - Specific technical decisions
- **Troubleshooting** - Past issues and solutions
- **Context** - Why certain approaches were taken

## What's Archived

### `/root-docs/`
Original documentation files from the project root:
- Architecture documents
- Setup guides
- OAuth implementation docs
- Deployment guides
- Phase completion summaries

### `/features/`
Individual feature implementation documents:
- Context menu implementation
- Landing page design
- Layer controls
- Layout builder
- Library modal
- Media pool enhancements
- PDF export progress

### `/fixes/`
Bug fix documentation:
- Dark mode fixes
- CORS fixes
- Frontend bugfixes

### `/migration/`
Migration guides and completion docs:
- Storage migration
- Auth system migration

## Using Archived Docs

When referencing archived documentation:

1. **Check current docs first** - The main docs are the source of truth
2. **Cross-reference** - Archived docs may be outdated
3. **Extract useful info** - Pull relevant details into current docs
4. **Don't link to archives** - Link to current docs instead

## Maintenance

- **Don't delete** - These docs provide valuable context
- **Don't update** - They're historical snapshots
- **Do consolidate** - Extract useful info into current docs
